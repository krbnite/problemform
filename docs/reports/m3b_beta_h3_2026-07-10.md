# H3 — Cross-Type Formulation Validation

## Objective

**Hypothesis (H3):** the **shipped v0.3.0-beta.1 evaluation framework**, run in its
**default configuration** as a user actually runs it, produces **coherent,
discriminating, and interpretable** results **across the full diversified benchmark
corpus (11 formulation types)** — i.e. ProblemForm evaluates *formulations*, not merely
questions. The universal `formulation_quality_v1` rubric is the central component under
test, alongside the answer-side rubric, property suite, and β.1's per-type answer-lens
policy.

This is an empirical validation of the existing implementation, not an improvement
task. No code was changed. (One truncated answer generation occurred — an OpenAI
answer-side truncation, not a framework defect; see Results — so it did not invalidate
the experiment.)

## Experimental Setup

- **Benchmark suite:** `benchmarks/cases/` — the canonical corpus, run as a **single
  `problemform benchmark` invocation** so mixed-type behavior and β.1's policy
  transitions are exercised exactly as a user sees them. (`benchmarks/simple/` excluded
  by design — an experimental scratch suite, not part of the canonical corpus.)
- **Formulation types (24 cases):** question 5, argument 1, and belief / decision /
  dilemma / explanation / goal / instruction / plan / prompt / specification at 2 each.
- **Providers / models (cross-family):** ProblemForm `gpt-4.1`, Answer `gpt-4.1`,
  Judge (rubric + comparative) `claude-sonnet-4-6`.
- **Rubric suites (default):** `formulation_quality_v1` (`target=formulation`) **and**
  `answer_quality_v1` (`target=artifact`) — both shipped defaults, no `--rubric`
  override.
- **Property suites (default):** `artifact_baseline_v1` (`target=artifact`), no
  `--property-suite` override; per-case `expected_properties` also activate.
- **Answer-comparison policy:** β.1 **per-type default** (no `--answer-comparison`
  override). Answerable types (question, explanation, instruction, prompt,
  specification) run the M3A answer lens; formulation-only types (argument, belief,
  decision, dilemma, goal, plan) skip it.
- **CLI command (no evaluation overrides):**
  ```
  problemform benchmark benchmarks/cases \
    --pf-provider openai --pf-model gpt-4.1 \
    --answer-provider openai --answer-model gpt-4.1 \
    --judge-provider anthropic --judge-model claude-sonnet-4-6 \
    --output .problemform/eval_runs/h3
  ```
- **Runs:** K=1, single run per case. Raw `report.md` preserved under
  [`m3b_beta_h3_2026-07-10/`](m3b_beta_h3_2026-07-10/report.md); full `report.json`
  under `.problemform/eval_runs/h3/` (gitignored).

## Results

### Execution health
24 cases → **12 completed, 1 errored, 11 answer-skipped**
(`n_cases = n_completed + n_errored + n_answer_skipped` holds). The 11 answer-skipped
cases are **exactly** the 11 formulation-only cases — β.1's policy fired correctly in a
single mixed run, generating no answers and writing no answer artifacts for them. The
single error was an OpenAI `TruncatedResponseError` on the *refined answer* of one
specification case (`online_bookstore_database`); its **formulation rubric still scored
normally** (Δ +0.65), so the formulation instrument was unaffected. The Anthropic
rubric judge completed ~500 structured calls with **zero JSON-mode failures**.

### Formulation rubric (`formulation_quality_v1`) by type — raw → refined (Δ)
| Type | n | raw | refined | Δ | answer lens |
|---|---|---|---|---|---|
| question | 5 | 0.24 | 0.66 | **+0.42** | on |
| explanation | 2 | 0.25 | 0.65 | **+0.40** | on |
| instruction | 2 | 0.10 | 0.68 | **+0.58** | on |
| prompt | 2 | 0.17 | 0.57 | **+0.40** | on |
| specification | 2 | 0.15 | 0.73 | **+0.58** | on |
| argument | 1 | 0.45 | 0.70 | **+0.25** | off (skipped) |
| belief | 2 | 0.17 | 0.80 | **+0.62** | off |
| decision | 2 | 0.15 | 0.60 | **+0.45** | off |
| dilemma | 2 | 0.45 | 0.70 | **+0.25** | off |
| goal | 2 | 0.15 | 0.77 | **+0.62** | off |
| plan | 2 | 0.22 | 0.62 | **+0.40** | off |

**Every one of the 11 types shows a positive refined − raw delta** (+0.25 to +0.62),
with refined scores clustering in 0.57–0.80. The **raw floor is interpretable**:
structurally richer inputs start highest (dilemma 0.55/0.35, argument 0.45), bare
inputs lowest (goal "I want to be a better person" 0.05; decision "what should I do
tomorrow" 0.05; instruction 0.05). Smaller deltas track higher raw floors (argument,
dilemma), larger deltas track low floors (belief, goal).

> **Headline contrast — the central H3 finding.** Across the run the **formulation**
> rubric rose **+0.46** (raw 0.22 → refined 0.68, all 24 cases) while the answer-side
> **`answer_quality_v1` *fell* −0.06** (raw 0.82 → refined 0.76, over the 13 answerable
> cases). Refining the formulation did **not**, on average, improve the first answer —
> and on several cases it lowered it. This is close to a direct empirical validation of
> the Constitution's central premise: **ProblemForm is optimizing the *formulation*, not
> answer quality.** The two lenses move in *opposite* directions here, so answer quality
> cannot stand in for formulation quality — the framework must measure the formulation
> directly (which is exactly what it does).

### Per-criterion (all 24 cases) — raw → refined (Δ)
| Criterion | raw | refined | Δ |
|---|---|---|---|
| `central_claim_clarity` | 0.77 | 0.96 | +0.19 |
| `constraint_articulation` | 0.08 | 0.84 | **+0.76** |
| `meta_question_presence` | 0.15 | 0.73 | +0.58 |
| `assumption_surfacing` | 0.05 | 0.46 | +0.41 |
| `alternative_framing_coverage` | 0.05 | 0.40 | +0.34 |

### Answer lens (answerable types) and disagreements
The M3A answer comparison (13 answerable cases) was a **weak, frequently null signal**:
mostly `tie/stylistic_only` or `*/minor`. The answer-side rubric aggregate **fell**:
`answer_quality_v1` raw 0.82 → refined 0.76 (**−0.06**), and in several cases the refined
*answer* scored markedly lower than the raw — e.g. `dragon_story` 0.85 → 0.35,
`clean_my_bedroom` 0.85 → 0.65, `code_review_prep` 0.90 → 0.75 (the judge's rationales
attribute these to refined prompts inducing longer, more procedural answers that gate or
bury the direct request). The report's disagreement diagnostic flagged **5 P3 cases**
("answer tie, large formulation gain", e.g. `web_app_money` Δ +0.60,
`cosmology_nothingness` Δ +0.50) — the formulation lens saw substantial improvement the
answer lens did not.

### Property lens
Property results appeared only for `question` (its 5 cases carry `expected_properties`)
and the four other answerable types (via `artifact_baseline_v1` against their answers).
**All six formulation-only types produced zero property results** — `artifact_baseline_v1`
is artifact-target (no answers to score) and those cases carry no `expected_properties`.
The property lens has no cross-type formulation coverage today.

### Representative examples
- **goal** `be_a_better_person` (0.05 → 0.85): raw "I want to be a better person." →
  refined names the ambiguity of "better", asks the user to clarify values/behaviors/
  outcomes, and lists sub-questions — a coherent, well-structured reformulation.
- **argument** `aquinas_narrative_designer` (0.45 → 0.70): raw already carries a claim →
  refined states the thesis ("challenge the view that Aquinas definitively proves God"),
  contrasts positions, and surfaces the underdetermination point.

## Analysis

**Does `formulation_quality_v1` generalize across formulation types?** On this
evidence, **yes**. It produced coherent, complete per-criterion scores for all 11 types
(no garbage, no incoherence), discriminated raw from refined in **every** type
(positive delta throughout, no sign flips), and its raw-floor pattern is interpretable
(it recognises pre-existing formulation structure in arguments/dilemmas and its absence
in bare goals/decisions). The universal rubric behaves as a cross-type instrument, not
a question-only one.

**Strongest criteria.** `constraint_articulation` is the strongest discriminator
(0.08 → 0.84, Δ +0.76) — ProblemForm reliably adds success criteria / constraints /
boundaries across types. `meta_question_presence` (Δ +0.58) is also strong.
`central_claim_clarity` scores highest in absolute terms (0.96 refined) but has little
headroom (raw already 0.77), so it discriminates least.

**Weakest criteria.** `assumption_surfacing` (refined 0.46) and
`alternative_framing_coverage` (refined 0.40) are weakest — even *refined* formulations
only reach ~0.4 on these. This is consistent across types and echoes H1/H2's finding
that assumption-surfacing is the least-improved dimension. It is unclear whether this
reflects a genuine ProblemForm weakness, a rubric-phrasing issue, or a judge tendency;
it is a candidate for calibration.

**Consistently difficult types?** None failed or scored incoherently. `argument` (n=1)
and `dilemma` show the smallest deltas, explained by their high raw floor (they already
carry a claim and competing considerations), not by rubric difficulty. `specification`
hit an answer-side truncation but its formulation scoring was clean.

**Candidates for type-specific rubrics.** The universal rubric already discriminates, so
type-specific rubrics are additive rather than urgent. The clearest opportunities: the
two weak criteria could be tuned or supplemented (e.g. argument-specific
premise/inference criteria; goal/decision-specific success-criteria emphasis). Evidence
for bespoke rubrics is suggestive, not compelling, at this corpus size.

**Candidates for type-specific property suites.** This is the **clearest gap**: six
formulation-only types have **no property signal at all**, because the default property
suite is artifact-target and those cases lack `expected_properties`. Formulation-target
property suites per type (or per-case `expected_properties` on the β cases) would give
the property lens cross-type coverage.

**Is β.1's answer-lens policy justified by the evidence?** **Yes.** For the 11
formulation-only cases the answer lens was correctly skipped (no natural answer). For
the 13 answerable cases where it *did* run, it was a weak/noisy signal — frequently a
tie, and sometimes a *regression* in answer quality even as the formulation improved
(the P3 disagreements). This reinforces the scope-note confound (answer quality is a
poor proxy for formulation quality) and supports gating the answer lens rather than
treating it as the headline for non-question types.

## Secondary Findings

Beyond the core hypothesis, H3 surfaced several findings worth recording in their own
right (each is developed above):

1. **Improving a formulation does not reliably improve the first answer.** Formulation
   +0.46 vs answer-quality −0.06 in the same run — the strongest single result, and a
   direct empirical echo of the Constitution (see *Headline contrast*).
2. **`answer_quality` is a weak proxy for formulation quality.** The two lenses
   frequently disagreed (5 P3 cases) and moved in opposite directions on aggregate; the
   M3A answer verdict should not be read as a formulation-quality signal.
3. **β.1's answer-lens gating is empirically justified.** The lens was correctly skipped
   for the 11 formulation-only cases, and where it *did* run it was weak/noisy and
   sometimes regressive — so gating it (rather than headlining it) is the right default.
4. **Constraint articulation is ProblemForm's strongest, most consistent refinement
   behavior** (criterion Δ +0.76, raw 0.08 → refined 0.84), ahead of meta-question
   presence; it adds success criteria / constraints / boundaries reliably across types.
5. **The property lens has a major blind spot for formulation-only types.** Six of
   eleven types received *no* property signal (the default suite is artifact-target and
   those cases carry no `expected_properties`) — the clearest actionable gap H3 found.
6. **Anthropic completed ~500 structured rubric/property evaluations with zero JSON-mode
   failures** — reassuring for using it as a structured judge at this scale (the single
   run error was an OpenAI answer-generation truncation, not a judge failure).

## Threats to Validity

- **Small, imbalanced corpus.** 24 cases; `argument` n=1, most types n=2, `question`
  n=5. Per-type deltas rest on 1–2 cases and are **directional only**, not statistically
  meaningful. Aggregate per-criterion numbers weight questions more heavily.
- **K=1, single run.** Each case scored once. H1 established the rubric is near-
  deterministic on fixed text, but refined-side scores still carry ProblemForm
  stochasticity; a single run cannot separate the two here.
- **Synthetic / hand-authored cases.** The corpus is author-constructed and may not
  represent real-world formulations; there is a latent risk of selection bias toward
  cases where ProblemForm helps. Partial mitigations: the control question and the
  observed answer-quality *regressions* show the framework is not uniformly flattering.
- **Single judge; not cross-validated.** One rubric judge (`claude-sonnet-4-6`); no
  intra-H3 replication or second-family judge. The judge differs from H1's `gpt-4o`, so
  cross-experiment magnitudes are only qualitatively comparable. (Same-family
  self-preference is not a concern here — answers `gpt-4.1`, judge `claude-sonnet-4-6`.)
- **One answer truncation.** The specification answer truncation drops answerable
  completed cases to 12 and is a reminder that long-artifact types can hit token
  limits — an answer-lens robustness note, not a formulation-lens issue.
- **Formulation-type imbalance** limits any claim about *relative* difficulty across
  types.

## Conclusions

On this small, imbalanced corpus, **H3 is supported**: the shipped v0.3.0-beta.1
framework — run with its default configuration — produces coherent, discriminating, and
interpretable formulation-quality results across all 11 formulation types. The universal
`formulation_quality_v1` rubric registered a positive, interpretable raw→refined
improvement for every type, with a consistent per-criterion structure, and the
representative refined formulations are sensible. This is concrete evidence that
**ProblemForm evaluates formulations, not merely questions.**

Two qualifications keep the claim proportional: (1) the corpus is too small and
imbalanced for statistical conclusions — the result is a strong *directional* signal,
not a measured effect size; and (2) the finding is coherence/discrimination, not
*correctness* — H3 does not establish that the rubric's judgments match expert human
judgment, only that they are internally coherent and type-general. The answer lens, by
contrast, is a weak proxy that sometimes contradicts the formulation lens, which both
justifies β.1's gating and cautions against over-reading M3A verdicts for non-questions.

Taken with its predecessors — H1 validated the rubric on questions, H2 showed it
generalized to a non-question — **H3 marks the transition from demonstrating that
ProblemForm can refine formulations to demonstrating that ProblemForm possesses a
coherent, formulation-centric evaluation framework spanning multiple classes of human
formulations.**

## Recommendations

**β.2 (make every formulation type measurable).** H3 experimentally identified the
current blind spot in the evaluation framework: **formulation-only types receive
essentially no property-level signal**, and their default artifact-side rubric
contributes nothing (both skip on empty answers). β.2 should therefore be motivated by
that evidence — its goal is that **every formulation type is meaningfully measured**,
via appropriate default rubric/property selection per type. The implementation
mechanism is the one already built for this: grow the `FormulationPolicy` registry with
`default_rubrics` / `default_property_suites` fields so answer-side defaults become
answerable-only, the universal `formulation_quality_v1` remains the baseline for all
types, and formulation-only types gain formulation-target property coverage. The
registry is the *how*; closing the measurability gap is the *why*.

**β.4 (type-specific rubrics / property suites).** Highest-value item: **type-specific
formulation *property* suites** (or per-case `expected_properties` on β cases) to close
the zero-coverage gap for the six formulation-only types. Lower-priority, evidence-gated:
consider tuning or supplementing the two weak rubric criteria (`assumption_surfacing`,
`alternative_framing_coverage`), and adding a small number of type-specific criteria
where a type has a distinctive success condition (e.g. arguments → premise/inference
explicitness). The universal rubric already works, so treat bespoke rubrics as additive.

**Future benchmark expansion.** Rebalance and enlarge: raise `argument` above n=1 and
add several cases per type for even directional stability; add at least one control
case per type (a well-formed input where refinement should barely move the score, to
guard against advocacy drift); include some real (non-synthetic) formulations; and
backfill `expected_properties` on the β-type cases so the property lens has cross-type
signal. A future run should also add K>1 and/or a second-family judge to separate rubric
noise from ProblemForm stochasticity and to check judge robustness.
